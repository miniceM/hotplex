## 🟢 核心理念：我们需要做什么？
要让 HotPlex 连上你的 Slack App，底层逻辑就是这两步：
1. 我们必须在 Slack 开放后台“**注册生成一个专属机器人**”。
2. 然后拿到这个聊天机器人的 **API 密钥（Tokens）** 填到你的 HotPlex 配置文件里。

就这么简单！我们开始吧！

---

## 第 1 步：一键创建你的 Slack 机器人

我们不需要一个一个去配置头像、名字和权限细节，直接用现成的魔法代码生成！

1. **登录大本营**：在浏览器打开并登录 [https://api.slack.com/apps](https://api.slack.com/apps)。
2. **点击绿色大按钮**：点击右上角醒目的绿色按钮 **"Create New App"**。
3. **选择创建方式**：
   - 弹出的窗口里，**必须选择第二个 "From an app manifest"**（意思是通过预设配置一键创建）。

![步骤 1：通过 Manifest 创建](../images/guides/slack_step1_manifest.png)

4. **选择工作区**：在下拉列表选中你们当前团队所在的 Workspace（比如 `BigTechCompany`），点击 **"Next"**。
5. **粘贴“魔法代码”**：
   - 此时你会看到一个满是代码的文本框（注意：上方标签选 "JSON"）。
   - **把框里原有的代码全部删除清空**。
   - **前往 👉 [这里（高级手册中的“应用配置清单”一节）](./chatapps-slack-manual-zh.md#⚡-应用配置清单-app-manifest) 复制那一段完整的 JSON 代码，原封不动地粘贴到你的框里！**
   - 点击右下角的 **"Next"**。
   - 确认无误后，再点击 **"Create"**。
   - 🎉 **恭喜第一步完成！机器人账号已经在后台建好了！**

---

## 第 2 步：开启环境互通通道 (Socket Mode)

为了让内网/本地运行的 HotPlex 也能穿透防火墙直接连上 Slack（无需公网 IP），我们必须打开 Socket 模式。

1. **回到左侧菜单**：在此应用详情页的左侧长条菜单栏中，一直往下找，找到并点击 **"Socket Mode"**。
2. **拨动开关**：把页面上的 **"Enable Socket Mode"** 的开关按钮拨到右侧 **"On" (🟢)** 状态。

![步骤 2：启用 Socket Mode](../images/guides/slack_step2_socketmode.png)

3. **给你的密钥起个名字**：
   - 系统会立刻弹出一个框，要求你生成一个底层通讯令牌（App-Level Token）。
   - 在 `Token Name` 框里随便填个名字，方便你自己认，比如：`hotplex_socket`。
   - 点击绿色的 **"Generate"**。
4. **复制第一把钥匙 (App Token)**：
   - 新弹出的框里会显示一串以 `xapp-...` 开头的超长字符串。
   - **点击 "Copy" 将它复制下来**，暂时粘贴在你的本地电脑记事本上（非常重要，马上就用！）。
   - 复制好后，点击 "Done" 关闭窗口。

---

## 第 3 步：把机器人安装到群组并拿第二把钥匙

目前这个机器人还只是存在于开发者后台的“温室”里，我们需要把它正式安装到你们员工聊天的频道中。

1. **去安装页面**：依然在左侧菜单栏，向上滚动，点击 **"Install App"**。
2. **点击安装按钮**：点击页面上唯一显眼的绿色按钮 **"Install to Workspace"**。
3. **检查并授权**：页面会跳转并询问你是否允许“HotPlex”访问你的团队资料并发送消息，直接点击绿色的 **"Allow"** 同意授权。
4. **复制第二把钥匙 (Bot Token)**：
   - 授权成功后跳回页面，你会看到一个名为 **"Bot User OAuth Token"** 的栏目。
   - 这串密码以 `xoxb-...` 开头。

![步骤 3：获取 Bot Token (xoxb)](../images/guides/slack_step3_tokens.png)

   - **点击 "Copy" 将它复制下来**，也粘贴到你刚才的本地记事本里。

---

## 第 4 步：告诉 HotPlex 你的接头暗号 (.env 配置)

现在你的记事本里应该有两段字符串了：
- 一段是 `xapp-` 开头的 （它是 App Token，代表 Socket 通道密码）。
- 一段是 `xoxb-` 开头的 （它是 Bot Token，也就是机器人的身份证）。

接下来我们要把它们写进 HotPlex 这个服务里：

1. **找到配置文件**：打开你存放 HotPlex 服务器代码的文件夹根目录。找到名为 `.env` 的文件。
   *(补充：如果找不到，就把旁边同名的 `.env.example` 文件复制一份并且重命名为 `.env`。)*
2. **填入 Token**：用文本编辑器（如 VSCode 或记事本文本编辑器）打开 `.env`，按照下面的格式填入你的密码：

```env
# 首先确保你的通信模式处于 socket 状态
HOTPLEX_SLACK_MODE=socket

# 填入你刚才拿到的【第二把钥匙】 (务必以 xoxb- 开头)
HOTPLEX_SLACK_BOT_TOKEN=xoxb-在这里粘贴你复制的内容

# 填入你刚才拿到的【第一把钥匙】 (务必以 xapp- 开头)
HOTPLEX_SLACK_APP_TOKEN=xapp-在这里粘贴你复制的内容
```

3. **保存文件**（注意一定要保存）。

---

## 第 5 步：启动并跟 AI 打招呼！

1. **启动 HotPlex**：在你的 HotPlex 服务器所在控制台/终端，敲下启动命令（比如 `make restart` 或其他启动指令）。只要控制台不疯狂闪红色的报错信息，就说明桥梁已顺利搭通！
2. **回到 Slack 客戶端聊天测试**：
   - 打开你们公司平时聊天的 Slack 软件或网页版。
   - 建立一个频道或随便找个测试频道。
   - **关键操作**：在这个频道里发送 `@HotPlex`。因为机器人刚装好还没进你的这个频道，Slack 会提示问你要不要把它拉进来，点击 **Invite them** 邀请进群。
   - 现在，你可以对着频道输入指令啦：敲击键盘上的 `/` 斜杠键，如果你能看到列表中弹出了 `/reset` 或 `/dc` 等快捷指令，恭喜你，大功告成！！！
   - 试着在群里或者点开它的私聊页面，对它说一句："Hi"，开启你的 AI 生产力之旅吧。
